Setup Prometheus
=================

Q. Why Prometheus? What are the requirement of using micrometer with Prometheus?
Q. How to setup Prometheus?

Steps
======

1. Add the micrometer dependency to the "pom.xml" file of all microservices, and load the mavne changes.
=>
<dependency>
    <groupId>io.micrometer</groupId>
    <artifactId>micrometer-registry-prometheus</artifactId>
</dependency>

2. Add the configuration to the "application.yaml" file of all microservices.
=>
management:
  endpoint:
    web:
      exposure:
        include: health,info,prometheus # This configuration will depend on your project requirements.
  metrics:
    tags:
      application: ${spring.application.name}

3. Test the metrics from the browser.
=>
Once you have added the dependencies and configuration, you can go to the browser and access the metrics endpoint.
http://localhost:8080/actuator/prometheus  # This will show the metrics in Prometheus format.
http://localhost:8080/actuator/metrics     # This will show the metrics in the default format.
http://localhost:8080/actuator/metrics/jvm.memory.used # This will show the metrics for a specific metric.

and so on, Test and play with it.


## Setup Distributed tracing using OpenTelemetry, Tempo and Grafana

1. add the open telemetry version and dependency.

<otelVersion>2.11.0</otelVersion> # or use stable version based on your project.

<dependency>
    <groupId>io.opentelemetry.javaagent</groupId>
    <artifactId>opentelemetry-javaagent</artifactId>
    <version>${otelVersion}</version>
    <scope>runtime</scope>
</dependency>

2. Add this logging pattern

logging:
  pattern:
    level: "%5p [${spring.application.name},%X{trace_id},%X{span_id}]"
    # %5p = log level, like - INFO, DEBUG etc.
    # ${spring.application.name} = application name
    # %X{trace_id} = trace id, which will be generated by OpenTelemetry and will be same for all the microservices for any request.
    # %X{span_id} = span id, which will be generated by OpenTelemetry and will different for each microservice but same for any particular microservice
    # this will help to identify the trace and span id in the logs.

3. Go to the docker compose file and add the following environment variables.

services:
  service_name:
    # Add this environment variable to all the services. or you can create a common yaml file and define
    # service and can use it from all the services by extending it.
    environment:
      OTEL_SERVICE_NAME: service_name
      OTEL_EXPORTER_OTLP_ENDPOINT: http://tempo:4318
      OTEL_METRICS_EXPORTER: none
      OTEL_LOGS_EXPORTER: none
      JAVA_TOOL_OPTIONS: "-javaagent:/app/libs/opentelemetry-javaagent-2.11.0.jar"

4. Add tempo configuration.

/observability/tempo/tempo.yaml <= this file will be used to configure tempo.
configure the tempo file as per your requirement.

add tempo service to the docker compose file and mention the datasource details for the grafana.
